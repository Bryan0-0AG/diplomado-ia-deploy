Guía y prompt maestro para crear una aplicación de 
Machine Learning con chatbot
Notebook en Google Colab + modelo entrenado + base limpia + documento del proyecto + interfaz Gradio
Propósito de esta guía
Orientar a los estudiantes para que construyan una aplicación web sencilla que combine dos elementos: un simulador de clasificación de  
salarios y un chatbot capaz de responder preguntas sobre el proyecto, el modelo, la base de datos y el documento académico. La app debe 
ejecutarse en Google Colab y publicarse mediante un enlace temporal de Gradio.
1. QUÉ DEBEN CONSTRUIR
Los estudiantes deben crear una aplicación en Python que funcione en Google Colab y tenga una interfaz visual en Gradio. La  
aplicación debe permitir clasificar un perfil laboral como salario_bajo o salario_alto y, además, responder preguntas relacionadas con  
el proyecto de Machine Learning.
 La simulación debe usar el modelo entrenado o un pipeline equivalente entrenado desde la base limpia.
 El chatbot debe responder únicamente sobre el proyecto, la base de datos, el modelo, las métricas, las variables, las limitaciones y las 
posibles aplicaciones.
 Cuando el usuario pregunte algo fuera del proyecto, el chatbot debe decir que no puede responder esa pregunta y pedir que se formule 
una pregunta relacionada con el trabajo.

4. REQUISITOS FUNCIONALES DE LA APP
 Debe tener una pestaña o sección para el chatbot del proyecto.
 Debe tener una pestaña o sección para el simulador de clasificación.
 Debe permitir ingresar o seleccionar las variables del perfil laboral: work_year, experience_level, employment_type, job_title, 
employee_residence, remote_ratio, company_location y company_size.
 Debe mostrar la clase predicha: salario_bajo o salario_alto.
 Debe mostrar una explicación breve de la predicción sin afirmar que es una verdad absoluta.
 Debe incluir mensajes de advertencia cuando el usuario escriba valores fuera del rango o categorías que no aparecían en la base original.
 Debe responder de forma ordenada y clara las preguntas del usuario sobre el proyecto.
5. REQUISITOS VISUALES DE LA INTERFAZ
La interfaz no debe verse como un formulario plano. Se recomienda organizarla con pestañas, tarjetas, texto introductorio, ejemplos  
de preguntas y una salida gráfica sencilla.
 Usar Gradio Blocks para organizar mejor la pantalla.
 Incluir título, descripción corta y advertencia de uso académico.
 Usar pestañas: Chatbot del proyecto, Simulador de clasificación y Guía de uso.
 Usar tarjetas o recuadros en escala de grises para mostrar resultados importantes.
 Usar un gráfico simple de barras o torta para acompañar la clase predicha, pero sin presentarlo como certeza absoluta.
6. ASPECTOS QUE DEBEN CUIDAR
Cuidado Qué hacer correctamente
Fuga de información No usar salary_in_usd como variable de entrada si esa columna se usó para 
crear la etiqueta salario_bajo/salario_alto.
Categorías nuevas Permitir escribir valores nuevos, pero avisar que el modelo aprende mejor 
con categorías presentes en el entrenamiento.
Año fuera del dataset Si el usuario escribe 2026 y la base llega solo hasta 2022, mostrar una 
advertencia.
Porcentaje 100% No decir que el modelo está absolutamente seguro. Explicar que algunos 
modelos, como Árbol de Decisión, pueden mostrar probabilidades extremas 
por la hoja donde cae el caso.
Preguntas fuera del proyecto El chatbot debe negarse de forma amable y pedir una pregunta relacionada 
con el proyecto.
Ética Aclarar que la app es académica y no debe usarse para decisiones laborales 
reales sin validación adicional.

Actúa como un desarrollador experto en Python, Machine Learning educativo y Gradio.

CONTEXTO DEL PROYECTO
- Tipo de problema: clasificación supervisada binaria.
- Variable objetivo: clase_salario o nombre_clase_salario.
- Clases: salario_bajo y salario_alto.

- La etiqueta se construye usando la mediana de salary_in_usd como punto de corte.
- salary_in_usd NO debe usarse como variable predictora porque produciría fuga de información.
- Variables de entrada esperadas: work_year, experience_level, employment_type, job_title, 
employee_residence, remote_ratio, company_location y company_size.
- La app tiene propósito académico y exploratorio, no debe presentarse como una herramienta de decisión 
laboral real.
ARCHIVOS QUE VOY A SUBIR A COLAB
1. Un CSV limpio con los datos del proyecto.
2. Un archivo .pkl con el modelo entrenado, si está disponible.
3. Un PDF o texto del informe final del proyecto, para usarlo como base de conocimiento del chatbot.
4. Opcionalmente, el notebook original de entrenamiento.
LO QUE DEBE HACER LA APLICACIÓN
1. Instalar y cargar las librerías necesarias: pandas, numpy, scikit-learn, joblib, gradio y matplotlib.
2. Permitir cargar o leer el CSV limpio.
3. Intentar cargar el modelo .pkl. Si el modelo no carga por problema de versión o ruta, reentrenar un 
pipeline desde el CSV.
4. Crear o reconstruir la variable objetivo si no existe, usando la mediana de salary_in_usd.
5. Retirar salary_in_usd y las columnas de etiqueta de las variables predictoras.
6. Crear un preprocesamiento adecuado:
   - OrdinalEncoder para experience_level, company_size y remote_ratio.
   - OneHotEncoder para employment_type, job_title, employee_residence y company_location.
   - Pasar work_year como variable numérica.
7. Crear un simulador donde el usuario ingrese un perfil laboral y el sistema clasifique el perfil como 
salario_bajo o salario_alto.
8. Permitir seleccionar valores existentes, pero también escribir valores nuevos cuando sea necesario.
9. Si el usuario escribe valores no vistos en la base, mostrar una advertencia clara.
10. Si el usuario usa un año fuera del rango del dataset, mostrar una advertencia.
11. Mostrar el resultado de forma visual y clara, con una salida textual y un gráfico simple en escala de 
grises.
12. Crear un chatbot que responda preguntas sobre:
    - objetivo del proyecto;
    - base de datos;
    - variables usadas;
    - variable objetivo;
    - mediana salarial;
    - modelos entrenados;
    - mejor modelo;
    - accuracy;
    - matriz de confusión;
    - preprocesamiento;
    - fuga de información;
    - limitaciones;
    - aplicaciones;
    - ética y sesgo;
    - trabajos futuros;
    - cómo presentar el proyecto.
13. Si el usuario pregunta algo que no pertenece al proyecto, el chatbot debe responder:
    "Lo siento, no puedo responder esa pregunta porque no está relacionada con este proyecto. Puedes 
preguntarme sobre la base de datos, el modelo, las variables, los resultados o las limitaciones."
14. Diseñar la interfaz con Gradio Blocks, usando pestañas:
    - Chatbot del proyecto.
    - Simulador de clasificación.
    - Guía de uso.
15. La interfaz debe verse moderna, ordenada y atractiva, usando escala de grises, tarjetas, títulos 
claros y ejemplos de preguntas.
16. Al final, ejecutar demo.launch(share=True) para generar un enlace público temporal.
ENTREGABLE ESPERADO
Genera el código completo en formato de notebook de Colab, explicado por celdas, para que yo pueda 
ejecutarlo de arriba hacia abajo. El código debe ser robusto, comentado, fácil de entender para 
estudiantes y debe evitar errores comunes de carga de archivos, categorías nuevas y versiones del modelo.

10. MENSAJE FINAL PARA LOS ESTUDIANTES
Idea central
La aplicación no consiste solamente en hacer una interfaz bonita. Debe demostrar que se entiende el ciclo de vida de Machine Learning:  
datos, variable objetivo, entrenamiento, evaluación, despliegue y comunicación responsable de resultados. Un buen proyecto no solo  
predice; también explica sus límites.
